# Retto Roulette 추첨 연출 감독 명세

- 작성일: 2026-07-19
- 대상: 회전 룰렛, 다트 복권
- 목표: 결과를 뽑는 기능이 아니라 시청자가 숨을 멈추고 경계를 확인한 뒤 환호하는 방송 장면을 만든다.
- 상태: 0.8.0 핵심 구현 완료. 회전·다트 물리, 경계 클로즈업, 결과 안착, 방송 안전영역에 적용했다.

## 1. 연출 판단

현재 연출은 극적이지 않다. 원인은 효과의 양이 아니라 **결정적 순간을 화면이 붙잡지 못하는 구조**다.

현재 회전 룰렛은 마지막에 원판 전체를 `1.13 → 1.19배` 키우고 포인터를 흔든다. 이것은 카메라가 경계를 찍는 클로즈업이 아니라 원판이 부풀어 오르는 CSS 효과로 보인다. 다트도 원판 내부 rim만 `1.16배` 키우며, 탄착 직후에는 오히려 `1.04배`로 줄어 충격의 크기가 사라진다.

복권다운 장면은 다음 인과관계를 한 숏으로 보여 줘야 한다.

```text
결과 고정
  → 원판은 계속 움직임
  → 카메라가 12시 포인터/탄착점을 추적
  → 경계 양쪽 후보가 화면에 들어옴
  → 슬로우 모션으로 경계 통과 또는 탄착
  → 정지한 물리적 증거를 충분히 유지
  → 당첨자 이름 공개
  → 전체 당첨자 목록에 안착
```

결과 카드가 원판을 대신하면 안 된다. 시청자가 먼저 봐야 할 것은 `이 원판의 이 지점에서 결정됐다`는 증거이고, 이름은 그 다음이다.

## 3. 새 무대 구조

물리와 카메라를 같은 transform에 넣지 않는다.

```text
broadcast-stage                         고정 16:9 방송 캔버스
├─ camera-rig                           줌·팬·초점만 담당
│  └─ wheel-frame
│     ├─ rotor                          실제 회전각만 담당
│     │  ├─ slices
│     │  ├─ RETTO ROULETTE hub          원판과 함께 회전, 반드시 보존
│     │  └─ embedded-dart               탄착 뒤 rotor에 완전 결합
│     └─ fixed-pointer                  12시 판정 기준
├─ dart-flight-layer                    탄착 전 screen-space 다트
├─ boundary-names                       경계 양쪽 후보의 전체 이름
├─ winner-ribbon                        정지 증거 위에 겹치는 이름
└─ result-rail                          전체 당첨자, motion 중에는 무대 밖
```

- `rotor`는 회전만 한다. 줌 때문에 물리 원판이 부풀거나 다트가 분리돼 보이면 안 된다.
- `camera-rig`는 12시 포인터 또는 다트 탄착점을 중심으로 이동·확대한다.
- `winner-ribbon`과 `result-rail`은 rotor transform을 공유하지 않는다.
- 비행 다트는 충돌 프레임에 같은 화면 좌표의 `embedded-dart`로 한 번만 교대한다.
- 클릭 프레임에서 결과·후보 스냅샷·최종 회전각을 고정한다. 카메라 변화는 결과를 바꾸지 않는다.

## 4. 회전 룰렛: 경계선 포토 피니시

### 장면 의도

회전 룰렛의 하이라이트는 **최종 정지 직전**이다. 다만 매 회차를 같은 경계로 만들지 않는다. 결과 고정 뒤 착지를 `진입 경계 / 반대 경계 / 칸 내부`로 나누고, 선택된 물리 위치에 맞는 카메라만 사용한다.

수학적으로 정확히 경계 위에 세우면 어느 쪽인지 모호하다. 최종 위치는 항상 당첨 조각 내부에 두되, 화면에서는 `간발의 차`로 읽히게 한다.

- 일반 조각: 경계 안쪽 약 `1~5°`.
- 좁은 가중치 조각: 조각 폭의 `20~35%` 안쪽.
- 경계에 놓인 것처럼 보여도 포인터 중심선과 당첨색 사이에 최소 2px의 판정 여백을 남긴다.
- 진입 경계는 실제로 넘어 당첨자가 왼쪽에 오고, 반대 경계는 당첨 칸 안에서 넘기 직전에 버텨 당첨자가 오른쪽에 온다.
- 칸 내부 착지는 경계 클로즈업 없이 넓은 감속 화면에서 12시 포인터로 증명한다.
- 권장 분포는 진입 경계 30%, 반대 경계 30%, 내부 40%이며 동일 회차의 결과 선택 확률에는 영향을 주지 않는다.

### 샷 타임라인

`T`는 실제 원판 정지 시점이다. 실제 감속 길이가 달라도 카메라는 물리 이벤트를 기준으로 붙는다.

| 샷 | 시점 | 원판 물리 | 카메라·화면 | 소리 |
|---|---:|---|---|---|
| R0. Ready wide | 추첨 화면 진입 후 계속 | 0→780°/s를 0.8~0.9초에 가속, 이후 정속 | 원판 전체 1.0배. 결과 레일 숨김 | 낮은 모터음, 속도에 맞춘 피치 |
| R1. Commit | 클릭 0~0.15초 | 속도 유지, 결과·최종각 고정 | `이 클릭에 결과 고정` 배지만 0.5초 점등 | 짧은 lock click |
| R2. Continuous deceleration | 클릭 후~`T-1.1초` | 클릭 순간 각속도에서 단 한 번의 연속 감속 | 1.0→1.10배, 포인터 쪽으로 매우 천천히 팬 | 각속도에 따라 모터 피치 하강 |
| R3. Push-in | `T-1.1~T-0.45초` | 마지막 몇 조각을 통과 | 12시 기준 1.10→1.35배. 주변 UI 감광 | 경계 tick 간격이 늘어남 |

| R5. Stop hit | `T` | 당첨 조각 안쪽에서 속도 0. 역회전/튕김 금지 | 최대 확대 상태에서 80~120ms 카메라 미세 recoil | 무게감 있는 stop clack |
| R6. Proof hold | `T~T+0.9초` | 완전 정지 | 최소 0.9초 포인터·경계·당첨 조각 유지. 결과판 금지 | 180~250ms 의도적 정적 |

| R8. Dock | 이후 0.45~0.6초 | 정지 유지 | 이름 리본이 결과 레일 새 행으로 이동. 완료 후만 레일/버튼 활성 | soft chime |

### 중요한 구현 계약

1. `auto-whirl`, `boundary-approach`, `boundary-creep`가 각각 새로운 감속을 시작하지 않는다.
2. 원판은 하나의 연속 각도 함수로 정지하고, 실제 경계형의 `distanceToBoundary`가 임계값에 들어올 때만 경계 연출 이벤트가 발생한다.
3. 경계 진입 전에는 당첨 쪽 색·이름을 미리 강조하지 않는다.
4. 경계 양쪽 이름은 원판 SVG 축약 이름이 아니라 전체 이름 오버레이다.
5. 반대 경계 정지는 경계를 실제로 넘지 않으므로 `boundary-crossed`를 발생시키지 않고 `boundary-held`로 확정한다.
6. 확대 중 원판이 화면에서 의도적으로 일부 잘릴 수는 있다. 단, 12시 포인터·경계선·양쪽 후보 표지는 모두 보여야 하며 결과 열 때문에 수직으로 잘리면 안 된다.

## 5. 다트 복권: 고속 회전 중 탄착

### 장면 의도

다트의 하이라이트는 최종 정지 직전이 아니라 **고속 원판에 꽂히는 충돌 순간**이다.

```text
고속 회전 + 미리 보이는 흔들리는 조준점
  → 클릭 순간 조준 좌표와 원판 위상 동결
  → 그 좌표에 도달할 실제 조각으로 결과 확정
  → 클릭과 동시에 정면 발사
  → 화살 뒤를 따라 탄착점으로 전진
  → 원판 속도는 그대로인 채 화면만 극단적으로 확대
  → 탄착
  → 다트가 원판과 함께 회전한다는 사실을 확인
  → 탄착 후 감속
  → 박힌 지점이 12시에 돌아와 정지
  → 당첨자 공개
```

느려지는 것은 탄착 직전 **카메라의 시간감과 다트 접근 표현**이다. 원판 각속도는 충돌 프레임까지 절대 낮아지지 않는다.

### 샷 타임라인

| 샷 | 시점 | 원판·다트 물리 | 카메라·화면 | 소리 |
|---|---:|---|---|---|
| D0. Ready aim | 진입 후 계속 | 원판 0→1080°/s 가속 후 정속 | 원판 전체, 상단 범위에서 흔들리는 조준점을 항상 표시 | 모터 루프 |
| D1. Launch | 클릭 0~0.15초 | 마지막 렌더 조준점·원판 위상 동결 후 실제 탄착 조각 확정, 다트 즉시 발사 | 정면 후방 POV. 동결된 탄착점과 다트 축을 같은 중심선에 둠 | 발사 snap |

### 경계 탄착 규칙

- 다트는 모든 회차에 `경계!`를 띄우지 않는다. 클릭 시 물리적으로 계산된 실제 near-boundary 탄착일 때만 사용한다.
- 탄착부터 정지 1초 전까지는 `경계에 꽂혔다`는 색 정보만 보여 주며 이름을 노출하지 않는다.
- 정지 1초 전에 원판 라벨과 경계 후보 이름을 함께 공개하되 승자 강조는 정지까지 미룬다.
- 최종 당첨자 확인은 다트가 박힌 원판이 12시에 멈춘 뒤 이루어진다.
- 시각 랜딩은 결과가 선택된 뒤 당첨 조각 내부에서 생성하므로 당첨 결과를 바꾸지 않는다.

### 충돌 물리 계약

```text
impactRotation = clickRotation + cruiseVelocity × flightTime
winnerIndex    = sliceAt(screenImpactAngle - impactRotation)
finalDelta     = normalize(12시 각도 - screenImpactAngle) + 360 × coastTurns
finalRotation  = impactRotation + finalDelta
```

- 비행 시간은 약 1.0~1.3초의 고정 범위에서 정하고, 클릭 순간 현재 각속도로 예상한 impact 회전값을 결과 확정에 사용한다. 당첨자에 맞춰 impact 회전값을 다시 계산하지 않는다.
- 충돌 전 마지막 300ms의 원판 각속도 편차는 5% 이하여야 한다.
- 감속은 반드시 `DART_IMPACTED` 이후에 시작한다.
- 다트는 충돌 프레임에 한 번에 박히고 크기를 앞뒤로 튕기는 bite 동작을 하지 않는다.
- 다트는 충돌 뒤 rotor의 자식이며 별도 screen-space 회전을 적용하지 않는다.
- rotor와 embedded dart의 상대각은 정지할 때까지 0으로 유지한다.

## 6. 장면별 UI 가시성

| 장면 | 남길 것 | 숨길 것 |
|---|---|---|
| Ready | 원판, 중앙 허브, 짧은 회차 제목, 공정성 한 줄, 주 버튼 | 빈 결과판, 명단·기록, 긴 설명 |
| Result committed | 원판, `이 클릭에 결과 고정` 배지 0.5초 | 설정/명단 버튼, 긴 phase 설명 |
| Motion | 원판, 포인터/다트, 다인 진행 수 `1/3` | 브랜드 헤더, 결과판, prompt, 공정성 장문, 모든 보조 버튼 |
| Boundary/Impact | 포인터 또는 다트, 경계 양쪽 후보 | 나머지 모든 UI |
| Proof hold | 정지 원판, 포인터/다트, 당첨 조각 | 결과판, 다음 버튼 |
| Winner reveal | 정지 원판, 이름 리본 | 전체 결과판, 조작 버튼 |
| Dock | 원판, 이동 중 이름, 결과 레일 | 다음 행동은 아직 숨김 |
| Completed | 화면에 맞는 원판, 최신 당첨자, 전체 당첨자, 다음 행동 | 반복 상태 문구, 무대 위 관리 도구 |

명단·기록 서랍은 공개 방송 캔버스를 덮으면 안 된다. 장기적으로는 진행자 제어 레이어와 OBS 송출 레이어를 분리한다.

## 7. 연출 이벤트 계약

물리 phase와 App의 화면 phase를 `onSpinEnd` 하나로 연결하지 않는다.

```text
STAGE_ENTER
CRUISE_REACHED
RESULT_COMMITTED
MOTION_STARTED
ZOOM_ENTERED
BOUNDARY_ENTERED
BOUNDARY_CROSSED
DART_LAUNCHED
DART_IMPACTED
DART_ATTACHED
BRAKE_STARTED
ROTATION_STOPPED
PROOF_HOLD_DONE
WINNER_REVEAL_STARTED
WINNER_REVEAL_DONE
DOCK_STARTED
DOCK_COMPLETED
```

- 원판이 `onRevealPhase({ revealId, phase, at, metadata })`를 App에 보낸다.
- App은 현재 `revealId`와 다른 오래된 이벤트를 무시한다.
- 카메라, 소리, 조명, UI는 같은 물리 이벤트를 구독한다.
- 고정 `setTimeout(4.96s)`로 물리 시점을 추측하지 않는다.
- 프레임 드롭이 있어도 각도·카메라·소리를 하나의 master timeline에서 보정한다.

## 8. 결과 공개와 전체 당첨자 안착

1. `ROTATION_STOPPED`와 `WINNER_REVEAL_STARTED`를 분리한다.
2. 원판을 최소 0.9초 보여 주기 전에는 이름을 크게 띄우지 않는다.
3. 이름 리본은 원판을 `opacity:0`으로 만들지 않는다.
4. 결과 레일 때문에 motion/reveal 도중 grid를 재계산하지 않는다.
5. Dock 시작 시에만 카메라가 원판을 6~10% 축소·이동한다.
6. 이름 리본의 실제 좌표에서 latest 결과 행까지 FLIP 이동한다.
7. 이름이 화면에서 사라지는 프레임 없이 결과 행으로 이어져야 한다.
8. Dock 완료 뒤에만 다음 행동을 표시한다.

## 9. 방송 해상도 계약

### 1280×720

- 전체 외곽 안전 여백 24px.
- motion 원판 기본 지름 430~500px.
- 경계 macro는 의도적 crop을 허용하되 포인터·경계·두 후보 이름을 보존.
- 경계 후보 이름 최소 28px, 당첨 이름 최소 48px.
- 하단 주 행동은 56px 안에서 항상 첫 화면에 표시.
- completed 원판 지름 360~420px, 결과 레일 340~400px.
- 페이지와 무대 모두 스크롤 금지.

### 1920×1080

- 1400px shell 제한을 라이브 무대에는 적용하지 않고 약 1840px까지 사용.
- motion 원판 지름 680~760px.
- 경계 후보 이름 38~48px, 당첨 이름 72~104px.
- completed 왼쪽 무대 약 1,220px, 결과 레일 480~540px.
- 결과 레일은 무대 상단 기준선과 맞추고 다음 행동까지 첫 화면에 포함.

## 10. 사운드와 조명

- 모터음·tick·다트 바람음의 속도는 실제 각속도/비행 진행률에서 파생한다.
- 경계 진입 시 배경을 약 15~20% 감광하고 다른 UI의 채도를 낮춘다.
- stop/impact 직후 180~250ms 정적을 넣어 충격을 만든다.
- 당첨 sting은 물리 정지 전에 재생하지 않는다.
- 회전: `lock click → 느려지는 tick → stop clack → 정적 → sting`.
- 다트: `launch snap → whoosh → impact thunk → 부착 회전 → stop click → 정적 → sting`.
- 음소거와 연출 음량을 즉시 제어할 수 있어야 한다.

## 11. 승인 기준

- 회전 룰렛의 최대 확대와 마지막 경계 통과 시점 차이가 100ms 이내다.
- 자동 룰렛의 감속 중 각속도가 phase 경계에서 다시 증가하지 않는다.
- 경계 macro에서 양쪽 후보 전체 이름을 최소 0.45초 동시에 읽을 수 있다.
- 다트 impact 전 마지막 300ms 동안 원판 속도 편차가 5% 이하다.
- 비행 다트와 embedded dart의 교대 위치 차이가 2px 이하이며 중복 프레임이 없다.
- 탄착 후 다트와 rotor의 상대각이 모든 프레임에서 고정된다.
- 정지 증거 구간이 0.9~1.2초다.
- 당첨 이름은 물리 정지 전에 노출되지 않는다.
- Hero가 떠도 포인터·당첨 조각·박힌 다트를 식별할 수 있다.
- Dock 중 이름이 화면에서 사라지는 프레임이 없다.
- 1280×720과 1920×1080에서 motion/reveal 중 결과판이 무대를 재배치하지 않는다.
- completed 화면에서는 원판이 결과 열 때문에 잘리지 않고 다음 행동까지 첫 화면에 보인다.
- `prefers-reduced-motion`에서도 `정지 증거 → 이름 공개 → 목록 안착` 순서는 유지한다.

## 12. 구현 순서

1. 물리 phase 이벤트 계약 추가.
2. `camera-rig`과 `rotor` transform 분리.
3. 자동 룰렛을 하나의 연속 감속 함수로 변경.
4. 경계 양쪽 후보 오버레이와 12시 macro camera 구현.
5. 다트 screen-space→rotor-space 충돌 교대와 짧은 attached proof 구현.
6. 0.9초 proof hold와 원판 위 winner ribbon 구현.
7. 실제 result docking과 완료 레이아웃 구현.
8. 장면별 UI 정리, 사운드 동기화, 720p/1080p 시각 회귀 테스트.

이 작업은 효과 몇 개를 더 붙이는 수정이 아니다. `원판 물리 → 카메라 → 결과 공개`를 하나의 감독 타임라인으로 재구성하는 작업이다.
