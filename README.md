# Retto Roulette

스트리머가 카페 댓글 참여자를 정리하고, 방송 화면에서 바로 공정하게 추첨하는 정적 웹 앱입니다.

## 추첨 상태와 방송 흐름

1. 첫 화면은 `새 추첨` 작업공간입니다. 당첨자/상품과 연출을 정하고, 오른쪽 샘플 원판에서 실제 방송 장면을 바로 확인합니다.
2. 사람 추첨에서 `명단 추가`를 누르면 현재 화면 위에 명단 서랍이 열립니다. 네이버 카페 글 전체를 붙여 넣어 댓글 작성자를 확인하거나, 한 줄에 한 명씩 직접 입력한 뒤 `Shift+Enter`로 한 번에 추가합니다. 저장 전 수정 내용도 미리보기 원판에 즉시 반영됩니다.
3. 명단이 준비되면 샘플이 실제 이름과 가중치로 교체됩니다. 동일 확률·당첨 후 제외가 기본이며, 이를 바꾸는 기능과 상품명·방송 제목·후보 일부 추리기는 `세부 설정`에 있습니다. 공개 연출은 `회전 룰렛`과 `다트 복권` 두 가지입니다.
4. `방송 화면 열기` 뒤에는 준비 패널을 접고 룰렛과 방송 행동만 크게 남습니다. 회전 룰렛과 다트 복권은 화면 진입 즉시 가속해 계속 회전하며, 약 0.9초 뒤 추첨 속도에 도달해야 결과 확정 버튼이 활성화됩니다.
5. 화면은 `준비 → 방송 → 결과`로 보이지만 내부에서는 `configuring → ready → locking → presenting → completed` 상태를 유지합니다. 한 번의 클릭은 한 결과만 고정하며, 결과 뒤 `한 명 더 뽑기`를 누를 때마다 같은 방송 보드에 공개 순서대로 누적됩니다.

## 초기화와 다시 시작

- `명단 바꾸기`는 기존 결과를 그대로 둔 채 명단 서랍을 엽니다. 취소하면 확정 결과로 돌아가고, 저장하면 새 추첨 작업공간으로 이동합니다.
- 명단 서랍의 `명단 비우기`는 참여자·당첨 제외·후보 풀만 비웁니다. 당첨 기록과 상품 설정은 보존합니다.
- `당첨 제외 상태 초기화`는 다음 회차에서 다시 뽑을 수 있게 할 뿐, 이미 공개된 결과나 CSV 기록을 바꾸지 않습니다.
- 첫 추첨 전과 결과 뒤에는 `추첨 종료 · 새로 설계`가 표시됩니다. 설계로 돌아가도 명단, 전역 당첨 기록, 당첨 제외와 차감된 상품 재고는 유지됩니다.
- 완료 뒤 `한 명 더 뽑기` 또는 `하나 더 뽑기`는 현재 방송 세션과 누적 결과 보드를 유지합니다. 설정 화면에서 다시 `방송 화면 열기`를 선택하면 새 방송 세션이 시작됩니다.
- 진행 도구의 `명단·상품·기록 모두 초기화`는 별도 확인 뒤 전체 데이터를 비웁니다. 방송 종료와 데이터 삭제는 서로 다른 행동입니다.

## 공정성 규칙

- 회전 룰렛은 멈추기 버튼을 누르는 순간 그 한 결과와 후보 규칙을 함께 확정합니다. 이후에는 확정된 결과를 공개만 합니다.
- 다트 복권은 발사 버튼을 누르는 순간 그 한 발의 결과와 정확한 후보 풀을 먼저 확정하고, 그 뒤에는 확정된 결과를 보여 주는 연출만 재생합니다.
- 사람은 `당첨 뒤 제외`를 끄면 다음 추첨에서 같은 사람의 중복 당첨을 허용합니다. 켜면 한 번 당첨된 사람은 다음 후보에서 빠집니다.
- 상품은 재고 한 개씩을 별도 추첨권으로 취급합니다. 같은 상품은 재고 수량까지만 나올 수 있고, 당첨되면 재고가 차감됩니다.
- 가중치는 모든 추첨의 결과 확률에 반영되며, 룰렛에서는 칸의 크기에도 비례해 표시됩니다. 가중치 `0`은 추첨에서 제외합니다.
- 각 기록에는 선정 시각, 공개 시각, 후보 수, 총 추첨권, 후보 지문(FNV-1a), 중복 규칙, 상품 원본/재고 단위가 남고 CSV로 내보낼 수 있습니다.

## 방송 연출

- 회전 룰렛은 추첨 화면 진입과 함께 0에서 빠르게 가속해 약 1,080°/s로 계속 돕니다. 가속 중에는 주 버튼을 잠가 저속 클릭의 재가속을 막고, 활성화된 버튼을 누르는 순간 결과가 고정됩니다. 이후 현재 속도에서 1차 제동한 뒤 실제 경계 앞의 별도 포토피니시로 넘어가 경계를 아슬아슬하게 통과하고 당첨 칸 안에 멈춥니다.
- 다트 복권도 진입 즉시 판이 고속 회전합니다. 버튼을 누르면 다트가 화면 정면에서 출발해 판 상단의 매번 달라지는 지점에 한 박자로 꽂히며, 착탄 전에는 판을 감속하지 않습니다. 실제 경계 탄착에서는 표지와 양쪽 후보를 정지까지 유지합니다. 꽂힌 다트는 판에 붙어 함께 돈 뒤 같은 경계가 12시에 왔을 때 당첨 칸과 당첨자를 확대·점멸해 공개합니다.
- 결과 보드는 같은 방송 세션에서 공개된 전체 당첨자를 누적하므로, 한 명씩 여러 번 뽑아도 이전 당첨자가 사라지지 않습니다.
- 방송 캔버스의 버튼은 상태별로 분리됩니다. 대기 중에는 추첨 시작이 유일한 주 행동이고, 완료 뒤에는 `한 명 더 뽑기`가 주 행동, `이번 추첨 끝내기 · 설계로`가 유일한 보조 행동입니다.
- 판 중앙의 `RETTO ROULETTE` 허브는 원판 내부에 있어 모든 상태에서 판과 같은 각도·속도로 회전합니다. 진행 중 제목은 12시 포인터를 비우고, 모바일 룰렛 크기는 화면 폭이 아니라 실제 캔버스 폭 안에서 제한됩니다.
- `명단 · 기록` 패널은 고정 오버레이이며 ESC, 포커스 이동, 닫은 뒤 원래 버튼으로의 포커스 복귀를 지원합니다. 회차 규칙 변경은 이 패널이 아니라 명시적인 `설정 바꾸기` 상태에서만 합니다.

마블 레이스는 현재 공개 UI에서 선택할 수 없습니다. 과거 `marble` 당첨 기록을 읽고 내보내는 호환성과 기존 엔진·스타일만 보존하며, 새 게임을 별도로 완성한 뒤 UI에 다시 통합합니다.

## 개인정보와 동작 방식

- 복사한 카페 내용은 사용자의 브라우저 안에서만 처리합니다.
- 네이버 로그인 세션, URL 수집, Cloudflare 프록시, 크롤링 서버를 쓰지 않습니다.
- 카페 원문 HTML이나 로그인 정보는 저장·전송하지 않습니다.
- 공개된 당첨 기록은 `retto-roulette-history`에 최대 100건 저장합니다. 추첨 버튼에서 이미 확정됐지만 아직 공개되지 않은 결과는 `retto-roulette-pending-result`에 먼저 저장하며, 새로고침 뒤 애니메이션을 재개하지 않고 `공개 전 확정 복구` 기록으로 합칩니다.

## 로컬 실행과 검증

```bash
npm install
npm test
npm run build
npm run dev
```

구조적 변경 전후에는 로컬 코드 그래프를 갱신합니다. `graphify-out/`은 커밋하지 않습니다.

```bash
graphify extract . --code-only
graphify query "raffle lifecycle and preview dependencies"
graphify update . --force
```

## CSS 관리 경계

- `src/rettoStock.css`는 레또스톡 공통 스킨의 동결된 기준 파일입니다.
- 공개 룰렛·다트와 보존 중인 마블 호환 컴포넌트의 레또스톡식 재스킨은 `src/styles/rettoRoulette.skin.css`에, 프로젝트 셸과 반응형 보정은 `src/styles/rettoRoulette.shell.css`에 둡니다.
- 준비 작업공간과 명단 서랍의 배치는 `src/styles/rettoRoulette.preparation.css`가 맡습니다. 화면 구조는 `src/App.css`, 개별 연출과 위젯은 각 컴포넌트의 CSS가 맡으며, 새 기능 스타일은 기준 파일을 수정하지 않고 이 경계 안에 추가합니다.

## GitHub Pages 배포

정적 빌드 결과를 `docs/`에 반영합니다. GitHub Pages 원본은 `main` 브랜치의 `/docs`이고, Vite 기본 경로는 `/rettoroulette/`입니다.

```bash
npm run pages:build
npm run pages:verify
# 푸시와 Pages 반영 뒤
npm run pages:health
```

상태 수명, 데이터 보존 범위, 승인·배포·롤백 절차는 [OPERATIONS.md](./OPERATIONS.md), 프로젝트별 작업 기준은 [CLAUDE.md](./CLAUDE.md)를 따릅니다.
