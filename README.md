# Retto Roulette

스트리머가 카페 댓글 참여자를 정리하고, 방송 화면에서 바로 공정하게 추첨하는 정적 웹 앱입니다.

## 추첨 상태와 방송 흐름

1. 네이버 카페 글에서 `Ctrl+A` → `Ctrl+C`한 내용을 붙여 넣어 명단을 만듭니다. 직접 입력은 곧바로 명단 다듬기로 갑니다.
2. 명단이 맞으면 `추첨 설정`에서 목적 → 후보/수령자 → 수량 → 연출 → 확률·중복 규칙 → 방송 제목 순서로 한 화면에서 정합니다. 모바일에서는 현재 설정과 추첨 화면 진입 버튼이 하단에 계속 보입니다.
   - 상품 추첨을 고르면 이름·수량·추첨권을 설정하는 상품 명단이 즉시 열립니다. 상품이 하나도 없을 때도 `첫 상품 추가`에서 막힘 없이 시작할 수 있습니다.
3. `이 설정으로 추첨 화면 열기` 뒤에는 설정을 접고, 룰렛·현재 회차 당첨자 보드·읽기 전용 규칙만 크게 남습니다.
4. 화면은 명시적인 상태로 움직입니다: `명단 준비 → 추첨 설정 → 추첨 대기 → 결과 고정 → 결과 공개 → 결과 확정`. 다트 복권 다인 추첨만 `다음 다트 대기`를 거칩니다.
5. 결과 확정 뒤에는 전체 당첨자를 먼저 확인한 다음 `남은 명단/같은 규칙으로 다음 추첨`, `규칙 바꾸기`, `명단 바꾸기`를 고릅니다. 후보나 재고가 바닥나면 주 버튼 자체가 초기화·보충 경로로 바뀝니다.

## 초기화와 다시 시작

- `명단 바꾸기`는 기존 결과를 그대로 둔 채 카페 페이지 붙여넣기 또는 직접 입력으로 새 명단을 준비합니다. 닫으면 확정 결과로 돌아갑니다.
- 명단 편집 화면의 `명단 비우기`는 참여자·당첨 제외·후보 풀만 비웁니다. 당첨 기록과 상품 설정은 보존합니다.
- `당첨 제외 상태 초기화`는 다음 회차에서 다시 뽑을 수 있게 할 뿐, 이미 공개된 결과나 CSV 기록을 바꾸지 않습니다.
- 다트 복권의 다음 다트 대기에서는 `이번 회차 여기서 마치기`를 선택할 수 있습니다. 공개된 결과와 재고/제외 반영은 유지되고, 회차만 명시적으로 종료됩니다.

## 공정성 규칙

- 자동 회전과 마블은 시작 버튼을 누르는 순간, 이번 회차의 모든 결과와 후보 규칙을 함께 확정합니다. 이후에는 결과를 순서대로 공개만 합니다.
- 다트 복권은 발사 버튼을 누르는 순간 그 한 발의 결과와 정확한 후보 풀을 먼저 확정하고, 그 뒤에는 확정된 결과를 보여 주는 연출만 재생합니다.
- 사람은 `당첨 뒤 제외`를 끄면 중복 당첨과 참가자 수보다 큰 당첨 인원을 허용합니다. 켜면 한 번 당첨된 사람은 현재 회차의 다음 후보에서 빠집니다.
- 상품은 재고 한 개씩을 별도 추첨권으로 취급합니다. 같은 상품은 재고 수량까지만 나올 수 있고, 당첨되면 재고가 차감됩니다.
- 가중치는 모든 추첨의 결과 확률에 반영되며, 룰렛에서는 칸의 크기에도 비례해 표시됩니다. 가중치 `0`은 추첨에서 제외합니다.
- 각 기록에는 선정 시각, 공개 시각, 후보 수, 총 추첨권, 후보 지문(FNV-1a), 중복 규칙, 상품 원본/재고 단위가 남고 CSV로 내보낼 수 있습니다.

## 방송 연출

- 자동 룰렛은 추첨 화면 진입과 함께 0에서 빠르게 가속해 약 780°/s로 계속 돕니다. 버튼을 누르는 순간 결과가 고정되고 현재 속도에서 재가속 없이 감속하며, 거의 멈춘 마지막 경계에서 확대·슬로우 하이라이트를 거쳐 당첨 칸 안에 멈춥니다.
- 다트 복권도 진입 즉시 판이 고속 회전합니다. 버튼을 누르면 다트가 화면 정면에서 출발해 가까워지는 판의 12시로 날아가고, 착탄 전에는 판을 감속하지 않습니다. 경계 탄착을 짧게 확인한 뒤 실제 다트가 판에 붙어 함께 돌고, 판이 당첨 칸 중앙에 멈췄을 때 최종 당첨자를 공개합니다.
- 결과 보드는 처음부터 작은 `0/N` 대기 상태를 보여 주고 첫 결과부터 확장됩니다. 5명은 한 화면, 6~10명은 2열로 보여 주며 새 당첨자는 자동으로 보이는 위치까지 이동합니다.
- 방송 캔버스의 버튼은 상태별로 분리됩니다. 대기 중에는 추첨 시작이 유일한 주 행동이고, 다트 대기에는 `다음 다트`와 현재 인원 마감만, 완료 뒤에는 다음 회차 행동만 보입니다.
- 판 중앙의 `RETTO ROULETTE` 허브는 원판 내부에 있어 모든 상태에서 판과 같은 각도·속도로 회전합니다. 진행 중 제목은 12시 포인터를 비우고, 모바일 룰렛 크기는 화면 폭이 아니라 실제 캔버스 폭 안에서 제한됩니다.
- `명단 · 기록` 패널은 고정 오버레이이며 ESC, 포커스 이동, 닫은 뒤 원래 버튼으로의 포커스 복귀를 지원합니다. 회차 규칙 변경은 이 패널이 아니라 명시적인 `설정 바꾸기` 상태에서만 합니다.

## 개인정보와 동작 방식

- 복사한 카페 내용은 사용자의 브라우저 안에서만 처리합니다.
- 네이버 로그인 세션, URL 수집, Cloudflare 프록시, 크롤링 서버를 쓰지 않습니다.
- 카페 원문 HTML이나 로그인 정보는 저장·전송하지 않습니다.

## 로컬 실행과 검증

```bash
npm install
npm test
npm run build
npm run dev
```

## CSS 관리 경계

- `src/rettoStock.css`는 레또스톡 공통 스킨의 동결된 기준 파일입니다.
- 룰렛·마블의 레또스톡식 재스킨은 `src/styles/rettoRoulette.skin.css`에, 프로젝트 셸과 반응형 보정은 `src/styles/rettoRoulette.shell.css`에 둡니다.
- 화면 구조는 `src/App.css`, 개별 연출과 위젯은 각 컴포넌트의 CSS가 맡습니다. `BroadcastActionDock`과 `PrizeEditor`도 독립 CSS를 사용하며, 새 룰렛 기능 스타일은 기준 파일을 수정하지 않고 이 경계 안에 추가합니다.
