# Retto Roulette

스트리머가 카페 댓글 참여자를 정리하고, 방송 화면에서 바로 공정하게 추첨하는 정적 웹 앱입니다.

## 방송 흐름

1. 네이버 카페 글에서 `Ctrl+A` → `Ctrl+C`한 내용을 붙여 넣어 명단을 만듭니다. 직접 입력은 곧바로 명단 다듬기로 갑니다.
2. 명단이 맞으면 방송 준비 화면에서 사람/상품, 인원, 중복 규칙, 가중치, 룰렛 연출을 한 번에 정합니다.
3. `방송 시작` 뒤에는 설정이 `진행 도구` 드로어로 접히고, 룰렛·현재 회차 당첨자 보드·읽기 전용 규칙만 크게 남습니다.
4. 이번 회차의 결과는 번호 순서대로 계속 보드에 남습니다. 새 회차를 시작하기 전까지 직전 결과는 바뀌지 않습니다.

## 공정성 규칙

- 자동 회전과 마블은 시작 버튼을 누르는 순간, 이번 회차의 모든 결과와 후보 규칙을 함께 확정합니다. 이후에는 결과를 순서대로 공개만 합니다.
- 양궁은 화살을 쏘는 버튼을 누르는 순간 그 한 발의 결과와 정확한 후보 풀을 확정합니다. 클릭 직후 화살이 출발하므로, 화면의 연출이 결과 선택보다 앞섭니다.
- 사람은 `당첨 뒤 제외`를 끄면 중복 당첨과 참가자 수보다 큰 당첨 인원을 허용합니다. 켜면 한 번 당첨된 사람은 현재 회차의 다음 후보에서 빠집니다.
- 상품은 재고 한 개씩을 별도 추첨권으로 취급합니다. 같은 상품은 재고 수량까지만 나올 수 있고, 당첨되면 재고가 차감됩니다.
- 가중치는 모든 추첨의 결과 확률에 반영되며, 룰렛에서는 칸의 크기에도 비례해 표시됩니다. 가중치 `0`은 추첨에서 제외합니다.
- 각 기록에는 선정 시각, 공개 시각, 후보 수, 총 추첨권, 후보 지문(FNV-1a), 중복 규칙, 상품 원본/재고 단위가 남고 CSV로 내보낼 수 있습니다.

## 방송 연출

- 자동 룰렛은 방송 화면을 채우는 큰 판으로 회전하고, 마지막 칸에서 확대·감속·하이라이트로 멈춥니다.
- 양궁은 고정된 1시 방향 탄착점으로 향합니다. 클릭 직후 발사, 화살 시점의 접근, 확대되는 판과 슬로우 착탄까지 한 흐름으로 공개합니다.
- 결과 보드는 처음부터 `0/N`과 대기 수를 보여 주고, 당첨자가 늘어도 방송 레이아웃이 흔들리지 않습니다.
- `진행 도구`는 고정 오버레이이며 ESC, 포커스 이동, 닫은 뒤 원래 버튼으로의 포커스 복귀를 지원합니다.

## 개인정보와 동작 방식

- 복사한 카페 내용은 사용자의 브라우저 안에서만 처리합니다.
- 네이버 로그인 세션, URL 수집, Cloudflare 프록시, 크롤링 서버를 쓰지 않습니다.
- 카페 원문 HTML이나 로그인 정보는 저장·전송하지 않습니다.

## 로컬 실행과 검증

```bash
npm install
npm test
npm run build
npm run dev
```
