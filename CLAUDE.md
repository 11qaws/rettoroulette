# Retto Roulette Project Rules

작업 전 `C:/Users/Qumin/.claude/CLAUDE.md`의 전역 규칙을 먼저 읽고, 이 파일의 프로젝트 경계를 함께 적용한다.

## 제품 목적과 기본 시나리오

- 이 앱의 사용자는 방송 중 추첨을 진행하는 스트리머이고, 시청자는 같은 방송 캔버스를 함께 본다.
- 핵심 결론은 `누구 또는 어떤 상품을 어떤 연출로 뽑았고, 방송 중 누적 결과가 무엇인가`이다. 필요한 수만큼 한 번에 하나씩 이어서 뽑는다.
- 첫 화면은 입력 폼이 아니라 준비 작업공간이어야 한다. 기본값만으로 추첨을 끝낼 수 있어야 하며 기본 순서는 `방송 제목 → 추첨 대상 → 연출 → 명단`으로 고정한다. 확률 지정과 후보 일부 추리기만 세부 설정에 둔다.
- 실제 방송 무대에는 준비 단계에서 확정한 방송 제목과 현재 원판 후보 스냅샷을 항상 표시한다. 후보는 원판 왼쪽의 읽기 전용 세로 명단이며 10개를 넘을 때만 느린 루프 스크롤을 사용한다. 방송 세션의 누적 수는 오른쪽 결과판 배지 한 곳에서만 시각적으로 표시한다.
- 현재 공개 연출은 회전 룰렛과 다트 복권뿐이다. 마블 선택지는 노출하지 않으며, 과거 기록 호환과 별도 게임 재구축을 위한 엔진·타입만 보존한다.
- 정상 상태의 장문 설명을 늘리지 않는다. 현재 상태, 실제 연출 미리보기, 바로 필요한 행동 하나를 우선한다.

## 상태와 공정성 경계

- 중심 상태는 `src/lib/raffleLifecycle.ts`의 전이표만 변경한다. 정의되지 않은 전이는 허용하지 않는다.
- 결과는 `locking`에서 먼저 확정하고 `presenting`에서는 이미 확정된 결과만 공개한다.
- 회전 룰렛의 경계 확대는 실제 경계 앞 포토피니시 구간에서만, 다트의 경계 표시는 실제 경계 탄착 순간에만 사용한다. 다트는 착탄 전 감속하지 않는다.
- 다트의 탄착 좌표 아래 조각과 최종 12시 당첨 조각은 같은 로컬 좌표·색이어야 한다. screen-space 탄착 표식은 원판이 다시 움직이는 순간 제거하고, 이후에는 원판에 결합된 다트만 접점을 표시한다.
- 라이브 원판이 0.9초 가속을 마치고 순항 속도에 도달하기 전에는 결과 확정·다트 발사 주 행동을 허용하지 않는다.
- 실행 중에는 명단·확률·재고를 편집하지 않는다. 늦은 애니메이션 콜백은 현재 `spinKey`와 `presentationRun`이 모두 일치할 때만 반영한다.
- `방송 화면 열기`에서 방송 세션을 만들고, 같은 설정으로 이어지는 회차의 공개 결과를 그 세션 보드에 누적한다. 한 회차 결과와 전체 방송 결과를 같은 배열로 대체하지 않는다.
- 구조적 상태 변경 전에는 `PREPARATION_WORKSPACE_SPEC_2026-07-19.md`를 갱신하고, 허용·금지 전이를 자동 테스트에 함께 추가한다.

## 데이터와 운영 범위

- 배포는 서버 없는 GitHub Pages 정적 앱이다. 네이버 로그인·크롤링·프록시를 다시 도입하지 않는다.
- 카페 원문과 명단은 브라우저 밖으로 전송하지 않는다. 공개된 당첨 기록은 기존 `retto-roulette-history` 키에 최대 100건 저장한다.
- 버튼 클릭에서 확정된 결과는 연출 전에 `retto-roulette-pending-result`에 기록한다. 정상 공개에서는 당첨 기록 저장이 성공한 뒤 pending 항목을 소비한다.
- 새로고침 뒤 현재 회차·방송 세션·애니메이션은 자동 재개하지 않는다. 유효한 pending 결과만 당첨 기록에 병합해 `공개 전 확정 복구`로 표시하고 CSV에 보존한다.
- 다중 탭이나 여러 진행자가 하나의 회차를 공동 편집하는 기능은 지원 범위가 아니다. 시청자는 스트리밍된 동일 캔버스를 본다.
- 상세 배포·점검·롤백 절차는 `OPERATIONS.md`를 따른다.

## 코드와 CSS 경계

- `src/rettoStock.css`는 동결한다. 프로젝트 전용 변경은 `src/styles/rettoRoulette.*.css` 또는 해당 컴포넌트 CSS에 둔다.
- 새 구조를 만들기 전에 기존 컴포넌트와 상태 모델을 재사용할 수 있는지 확인한다.
- Graphify 그래프가 없거나 구조 변경 뒤 오래되었으면 `graphify extract . --code-only` 또는 `graphify update . --force`로 갱신한다. `graphify-out/`은 로컬 산출물이며 커밋하지 않는다.
