# Retto Roulette 통합 준비 작업공간 명세

- 작성일: 2026-07-19
- 대상 버전: 0.10.4
- 상태: 구현 기준 확정
- 목표: 준비 작업공간의 공개 연출을 회전 룰렛·다트 복권으로 집중하고, 클릭 시점 결과 확정부터 포토피니시·방송 세션 누적 결과·새로고침 복구까지 한 흐름으로 검증한다.

## 1. 핵심 시나리오

스트리머는 Retto Roulette를 열어 다음 결론을 빠르게 얻으려 한다.

> 누구 또는 어떤 상품을, 몇 개, 어떤 방송 연출과 규칙으로 뽑을지 정하고 방송 화면을 연다.

명단은 이 결론을 위한 입력 데이터이지 제품의 첫 번째 목적이 아니다. 따라서 별도 명단 마법사는 폐지하고, 첫 진입부터 목적·입력·핵심 설정·미리보기·시작 행동을 같은 작업공간에 둔다.

## 2. 사용자에게 보이는 상태

| 표시 상태 | 내부 상태 | 주 화면 | 허용 행동 |
|---|---|---|---|
| 준비 | `roster`, `configuring` | 통합 준비 작업공간 | 대상 준비, 설정 변경, 미리보기, 방송 화면 열기 |
| 방송 | `ready`, `locking`, `presenting`, `awaiting-dart` | 방송 캔버스 | 결과 고정·공개, 진행 도구 확인 |
| 결과 | `completed` | 원판 + 방송 세션 누적 당첨자 | 같은 규칙의 다음 추첨, 규칙 변경, 명단 변경, 제외 복구 |

내부 `RaffleStatus`와 결과 확정/연출 분리는 유지한다. 앱은 `configuring`으로 시작하며, `roster`는 독립 페이지가 아니라 명단 편집 드로어가 열린 안전한 편집 트랜잭션으로 사용한다.

### 준비 상태의 보조 정보

중심 상태를 늘리지 않고 다음 값에서 화면을 파생한다.

- 목적: `people | prizes`
- 대상 준비: 사람 명단 수 또는 유효 상품 재고 수
- 편집 세션: 카페 붙여넣기, 확인, 직접 편집, 설정
- 준비 가능 여부: 유효 후보 수, 당첨 수, 가중치, 제외 상태
- 세부 설정 열림 여부: 화면 생애주기와 분리된 로컬 UI 상태
- 방송 세션: `방송 화면 열기`에서 생성한 ID, 추첨 대상, 공개 순서의 누적 결과
- pending 결과: 버튼 클릭에서 확정됐지만 아직 기록 저장이 끝나지 않은 결과

## 3. 준비 작업공간 전이

| 현재 | 이벤트 | 다음 | 불변식 |
|---|---|---|---|
| 사람·명단 없음 | 명단 추가 | 사람·명단 드로어 | 추첨 기록과 상품은 변경하지 않음 |
| 사람·명단 편집 | 명단 사용 | 사람·설정 | 명단 ID를 저장하고 제외 ID만 교집합으로 보존 |
| 사람·설정 | 명단 편집 | 사람·명단 드로어 | 저장 전까지 방송 후보는 변경하지 않음 |
| 준비 | 상품 선택 | 상품·설정 | 사람 명단은 보존하되 상품 추첨의 필수조건으로 요구하지 않음 |
| 상품·설정 | 사람 선택, 명단 있음 | 사람·설정 | 기존 사람 설정을 복원 |
| 상품·설정 | 사람 선택, 명단 없음 | 사람·준비 | 명단 추가 CTA로 드로어를 열기 전까지 시작 차단 |
| 설정 완료 | 방송 화면 열기 | `ready` | 이때의 설정을 다음 회차 초깃값으로 사용 |

명단 드로어가 열리면 배경의 목적·설정 컨트롤은 잠긴다. 초안은 저장 명단과 분리하고 `취소`에서는 폐기한다. 실행 중에는 목적·명단·확률을 바꿀 수 없다.

## 4. 1080p 정보 구조

기본 데스크톱 작업공간은 `1600~1700px` 폭을 사용하고, 브라우저 UI를 제외한 높이 약 `900px` 안에 맞춘다.

```text
브랜드 · 새 추첨 준비                         준비 / 방송 / 결과
┌ 설정 420~500px ──────────┬ 방송 연출 미리보기 ───────────┐
│ [당첨자] [상품]            │ [미리보기 · 기록 안 됨]        │
│ 명단 12명           [편집] │ 실제 이름과 확률이 반영된 연출  │
│ 당첨 인원    [−] 1 [＋]     │                              │
│ 연출        [룰렛][다트]    │ 12명 중 1명 · 동일 확률        │
│ [세부 설정 · 동일 · 당첨 제외]│ ✓ 준비 완료                   │
│                            │ [12명 중 1명 · 방송 화면 열기]  │
└───────────────────────────┴──────────────────────────────┘
```

### 항상 표시

- 사람/상품 목적
- 현재 유효 후보 수와 문제가 있을 때만 제외 수
- 당첨 수
- 연출
- 현재 확률·당첨 후 정책의 짧은 요약
- 준비 완료 또는 정확한 차단 이유 하나
- 동작 미리보기
- 결과를 말하는 주 행동 하나

사람 명단이 비어 있으면 해결 행동 `명단 추가`는 왼쪽 `명단` 행에 둔다. 오른쪽 아래 `방송 화면 열기`는 차단 이유를 반복하거나 다른 위치의 복구 행동을 대신하지 않고 중립 비활성 상태로 남는다.

### 기본 상태에서 접기

- 후보 일부 무작위 선택
- 개별 가중치 편집
- 상품명/이벤트명과 방송 표시 제목
- 전체 명단 개별 편집
- 상세 상품 재고 편집

고급 편집은 작업공간 내부 높이를 늘리지 않고 드로어 또는 제한 높이 패널에서 스크롤한다.

## 5. 문구와 시각 문법

- 정상 상태는 명사형 라벨과 현재값만 표시한다.
- `해요`, `하세요`, `할 수 있어요` 형태의 상시 설명을 제거한다.
- 오류는 원인 한 줄과 해결 행동 하나만 표시한다.
- 위험하거나 되돌리기 어려운 변경은 클릭 뒤 확인창에서 설명한다.
- 재미있는 문구는 방송 연출과 결과에만 사용하고 설정 화면은 담백하게 유지한다.
- 화면 안의 solid primary 버튼은 한 개만 둔다.
- 정보 요약은 중립 배경, 선택은 체크 표지가 있는 segmented control, 실행은 solid 버튼으로 구분한다.

기본 라벨은 `추첨 대상`, `명단`, `당첨 인원`, `연출`, `세부 설정`이다. 확률과 당첨 후 정책은 요약만 항상 보이고 변경 컨트롤은 펼친 뒤 표시한다.

## 6. 명단 드로어 계약

- 사람 추첨에서 명단이 없거나 편집을 선택했을 때만 작업공간 위에 드로어를 연다.
- 입력 방식은 `카페 댓글 | 직접 입력`을 동등한 탭으로 표시한다.
- 카페 입력의 상시 안내는 `Ctrl+A → Ctrl+C → 붙여넣기` 한 줄만 둔다.
- 파싱 뒤에는 `32명 · 대댓글 38개 제외`처럼 결과만 표시한다.
- 명단이 정확하면 `이 명단 사용`, 수정이 필요할 때만 `명단 수정`을 누른다.
- 붙여넣기·확인·수정은 별도 페이지로 넘기지 않고 같은 드로어 안에서 갱신한다.
- 직접 편집 목록은 드로어 내부에서만 스크롤하고 완료 버튼은 고정한다.
- 직접 입력은 일반 Enter로 줄을 나누고, `Shift+Enter`로 현재 여러 줄을 명단에 한 번에 추가한다. 한글 IME 조합 중에는 단축키를 실행하지 않는다.
- 초안의 이름·수·순서 변경은 오른쪽 미리보기에 즉시 반영한다.

## 7. 동작 미리보기 계약

미리보기는 실제 `RouletteWheel`과 `DartFinish` 시각 컴포넌트를 사용하되 추첨 데이터에는 영향을 주지 않는 별도 감독기가 소유한다.

### 입력

- 후보 이름
- 가중치
- 사람/상품 목적
- 당첨 수
- 회전 룰렛/다트 복권 연출

### 불변식

- 실제 난수 선택, 당첨 기록, 재고 차감, 당첨 제외를 실행하지 않는다.
- 샘플 결과 인덱스는 결정적으로 순환한다.
- `미리보기` 표지를 항상 표시하고 큰 Winner Hero는 띄우지 않는다.
- 설정 변경 후 250ms 이내 새 데이터로 주기를 다시 시작한다.
- 화면 밖·탭 비활성에서는 정지한다.
- reduced-motion에서는 정적 키프레임을 표시하고 수동 재생만 제공한다.

### 회전 룰렛

`가속(주 버튼 잠금) → 고속 회전(주 버튼 활성) → 결과·착지 고정 → 고속 제동 → 선택된 착지 연출 → 정지 증명 → 반복`

결과를 먼저 확정한 뒤 결과와 무관한 시각 착지를 `진입 경계 근처 / 반대 경계 근처 / 칸 내부`에서 각각 약 `30% / 30% / 40%`로 만든다. 진입 경계는 이웃 칸에서 당첨 칸으로 실제 통과해 당첨자가 화면 왼쪽에 오고, 반대 경계는 당첨 칸 안에서 경계를 넘기 직전에 멈춰 당첨자가 화면 오른쪽에 온다. 내부 착지는 칸 폭의 `24~76%`에서 정하고 경계 이름·표지·펄스를 띄우지 않는다. 좁은 가중치 칸은 항상 해당 칸 내부와 판정선 여백을 보장한다. 경계형에서만 고속 제동의 끝 속도와 최소 `1.45초` 포토피니시의 시작 속도를 맞춘다.

### 다트 복권

`고속 회전 + 움직이는 조준점 → 클릭 순간 좌표·원판 위상 고정 → 정면 발사 → 무감속 순간 탄착 → 다트/판 결합 회전 → 정지 1초 전 이름 공개 → 최종 증명`

`ready`와 `awaiting-dart` 진입마다 결과와 무관한 조준 세션을 하나 만든다. 탄착점은 원판 상단 `-115~-65°`, 반지름 비율 `0.58~0.80` 안에서 계속 부드럽게 흔들리며 발사 전부터 보인다. 클릭은 마지막으로 실제 그려진 조준 좌표와 현재 원판 각속도를 먼저 동결한다. 고정 비행 시간 뒤 그 화면 좌표 아래에 도달할 실제 원판 조각을 즉시 계산해 결과를 확정하고, 그 뒤 pending 저장과 연출을 시작한다. 따라서 당첨자에 맞춰 원판을 재조준하지 않으며 조각 크기로 표현된 동일·가중 확률을 그대로 사용한다.

동결한 `DartShotPlan`은 화면 다트·표적·원판에 박힌 다트·카메라 확대의 단일 좌표 원본이다. 비행 중 접근 지터는 탄착 직전에 0으로 수렴하고 실제 탄착점과 embedded dart에는 적용하지 않는다. 충돌 프레임에는 다트가 한 번에 나타나며 박힌 뒤 크기를 앞뒤로 튕기는 `bite` 동작을 사용하지 않는다.

| 다트 시각 상태 | 원판 속도 | 경계 정보 | 당첨자 강조 |
|---|---|---|---|
| `aiming` | 이름을 읽기 어려운 순항 속도 | 움직이는 조준점만 표시 | 숨김 |
| `launch / approach` | 이름을 읽기 어려운 순항 속도, 선형 | 동결된 조준점 | 숨김 |
| `impact` | 순항 속도 유지 | 실제 경계 탄착일 때 두 후보의 색만 표시 | 숨김 |
| `coast` · 정지 1초 이전 | 다트와 함께 감속 | 경계형은 두 색 유지, 이름 숨김 | 숨김 |
| 정지 1초 전 | 연속 감속 | 원판 흐림 제거, 원판과 경계 카드 이름 동시 공개 | 중립 |
| `settled` | 정지 | embedded dart가 도착한 최종 12시를 중심으로 확대해 1.5초 유지 | 실제 당첨 칸과 이름만 확대·점멸 |

탄착 contact는 화면 다트가 사라지고 같은 크기의 embedded dart가 한 프레임에 나타나는 한 박자로 보이게 한다. 경계 하이라이트는 실제 로컬 탄착점이 경계 임계거리 안이고 후보가 둘 이상일 때만 허용한다. `start` 경계에서는 당첨자가 오른쪽, `end` 경계에서는 당첨자가 왼쪽이며, 내부 탄착에는 경계 UI가 없다. 다트는 판에 박힌 뒤 판과 함께 돌며 같은 로컬 탄착점이 최종 12시로 이동하고, 회전 룰렛의 느린 정지 포토피니시를 재사용하지 않는다.

### 마블 호환 범위

마블 레이스는 0.10.0 공개 설정·미리보기·방송 진입 선택에서 제외한다. 기존 `MarbleRace`, `DrawMode: marble`과 과거 기록 읽기/CSV 호환은 삭제하지 않으며, 새 게임 스테이지를 별도로 구축하고 검증한 뒤 다시 통합한다.

## 8. 방송 세션과 결과 무대

- `configuring → ready`의 `방송 화면 열기`가 새 방송 세션을 만든다.
- `completed → ready`의 같은 규칙 다음 추첨은 세션 ID와 누적 결과를 유지한다.
- 설정으로 돌아가 다시 방송 화면을 열면 새 세션을 시작한다. 과거 결과는 전역 당첨 기록에 남지만 새 방송 보드에는 섞지 않는다.
- `CurrentRound.results`는 이번 회차의 진행·남은 수를 판정하고, `BroadcastSession.results`는 시청자에게 이번 방송의 전체 당첨자를 공개 순서대로 보여 준다.
- 결과 Hero는 원판 위 오버레이가 아니라 별도 결과 레일에 놓는다. Hero가 접힌 뒤 같은 자리의 누적 보드로 전환하며 원판의 바운딩 박스와 겹치지 않는다.
- 12시 포인터는 원판 크기에 비례한 작은 표지로 두고, 원판의 시각적 주도권을 침범하지 않는다.
- 12시 포인터는 위쪽 핀을 물리적 고정축으로 사용해 아래 침 끝만 흔들린다. 흔들림 종료 프레임은 반드시 `rotate(0deg)`이며, 이때 침 끝이 원판 중심과 같은 x축에서 최종 당첨 칸을 가리킨다.
- 경계 포토피니시는 경계 접근을 보여 주는 연출 좌표와 최종 판정을 증명하는 정지 좌표를 분리한다. 최종 정지에서는 구분선 두께보다 충분히 당첨 칸 안쪽에 들어오며, 좁은 칸은 칸 중앙을 사용한다.
- `end` 경계 클로즈업은 왼쪽 당첨자·오른쪽 다음 이웃, `start` 경계 클로즈업은 왼쪽 이전 이웃·오른쪽 당첨자로 실제 원판 좌우와 일치시킨다.

## 9. 오류와 복구

| 차단 원인 | 표시 | 해결 행동 |
|---|---|---|
| 사람 명단 없음 | `명단 없음` | 왼쪽 명단 행의 `명단 추가` |
| 전원 당첨 제외 | `후보 0명 · 4명 제외` | `제외 4명 복귀` |
| 모든 가중치 0 | `추첨권 0장` | `동일 확률로 전환` |
| 상품 없음 | `상품 없음` | `상품 추가` |
| 후보 일부 선택 결과 0 | `선택 후보 없음` | `전체 명단 사용` |
| 확정 결과 공개 전 새로고침 | `공개 전 확정 복구` | 당첨 기록/CSV에서 확인, 애니메이션은 재개하지 않음 |

정상 상태의 `제외된 사람 없음`, `남은 명단 전체 사용` 문구는 표시하지 않는다.

## 10. 반응형 기준

- 라이브 화면은 모니터 해상도가 아니라 브라우저 UI를 제외한 실제 `100dvh`를 기준으로 배치한다.
- 모든 라이브 상태(`ready`, `locking`, `presenting`, `awaiting-dart`, `completed`)에서 헤더·상태 바·액션 도크 높이를 먼저 확보하고, 남은 무대의 가로·세로 중 작은 값으로 원판을 정한다.
- 1920×1080 모니터의 약 1920×1000 페이지 뷰포트: 준비·연출·결과·다음 추첨 준비 모두 문서 스크롤 없이 주 CTA 전체가 보임.
- 1366×768: 목적, 후보 수, 당첨 수, 연출, 규칙, 준비 상태, CTA가 동시에 보이며 원판과 결과 레일은 액션 도크를 침범하지 않음.
- 데스크톱 폭에서 높이가 부족하면 설명 문구를 숨기고 원판을 축소한다. CTA를 화면 밖으로 보내거나 문서를 잘라 숨기는 방식은 금지한다.
- 900px 이하: 설정과 미리보기를 한 열로 바꾸고 문서 스크롤을 허용한다. 주 CTA는 미리보기 하단에 유지한다.
- 모바일에서 세부 설정만 내부 스크롤하며 가로 스크롤은 허용하지 않는다.

## 11. 검증 기준

- `document.scrollHeight <= document.clientHeight + 1` — 1080p/768p 기본 준비 상태.
- 기본 화면에서 solid primary 행동 하나.
- 모든 필수 설정이 스크롤 없이 보임.
- 명단 0명에서 왼쪽 명단 행의 `명단 추가`가 유일한 복구 행동이고, 우하단 CTA는 비활성임.
- 공개 연출 선택과 미리보기에는 회전 룰렛·다트 복권만 나타남.
- 명단·인원·가중치·연출 변경이 250ms 내 미리보기에 반영됨.
- 미리보기 원판의 transform이 시간에 따라 바뀜.
- 다트 탄착 뒤 embedded dart가 원판 회전 요소의 자식임.
- 다트 비행 표적과 embedded dart 중심 좌표가 탄착 프레임에서 일치함.
- 다트 경계 탄착에서는 `경계선!`과 두 후보가 impact → coast → settled 동안 끊기지 않고, 최종 12시에서 당첨 카드와 조각만 확대·점멸함.
- 서로 다른 다트 실행에서 탄착 좌표가 달라지며, 지터는 실제 접점에서 0으로 수렴함.
- 다트 대기 중 조준점이 보이고 움직이며, 클릭 직전 마지막 렌더 좌표와 동결된 발사·탄착·embedded dart 좌표가 일치함.
- 다트 결과는 클릭 시 동결한 원판 회전·각속도와 비행 시간으로 계산한 실제 탄착 조각이며, 당첨자에 맞춘 별도 원판 재정렬이 없음.
- 다트는 정지 1초 전까지 원판 이름과 경계 카드 이름을 숨기고 색만 유지한 뒤, 1초 전에 두 영역의 이름과 선명도를 동시에 공개함.
- 회전 룰렛은 고속 제동과 별도 포토피니시 구간을 거치며, 다트는 탄착 전 고속을 유지함.
- 회전·다트 모두 `near-start / near-end / interior`가 재현되며, 내부 착지에는 경계 UI가 없고 양쪽 경계의 후보와 당첨자 좌우가 원판 이동 방향과 일치함.
- 미리보기 한 주기 뒤 기록·제외·상품 재고가 변경되지 않음.
- 후보 0명에서 당첨 수가 순간적으로 `1명`으로 보이는 플리커 없음.
- 1920×1080·1366×768 방송 결과 화면에서 원판과 Hero/누적 결과 레일의 교차 면적이 0임.
- 1920×1000·1600×900·1366×768 데스크톱 뷰포트에서 첫 추첨 전, 결과 확정, 누적 결과가 있는 다음 추첨 대기의 액션 도크 하단이 항상 `innerHeight` 이내임.
- 라이브 무대의 상단·하단 설명 문구가 원판 림 또는 12시 포인터의 바운딩 박스와 교차하지 않음.
- 2~10명·가중치 명단·첫/마지막 인덱스에서 최종 원판 회전값으로 12시 아래 조각을 역산하면 잠긴 `winnerIndex`와 항상 같음.
- 포토피니시와 최종 증명 프레임에서 포인터 끝점 x좌표가 원판 중심 x좌표와 일치함.
- 포인터 흔들림 중 위쪽 핀 중심은 움직이지 않고 아래 침 끝만 움직이며, 종료 프레임의 침 끝은 12시 판정선으로 복귀함.
- 같은 방송 세션에서 한 명씩 두 회차를 마치면 누적 보드에 두 결과가 공개 순서대로 남음.
- 새 방송 세션에서는 누적 보드가 비고, 전역 당첨 기록은 보존됨.
- pending 결과는 history 저장 성공 뒤 소비되고, 새로고침 복구에서는 같은 ID를 중복 삽입하지 않음.
- 기존 라이브 추첨의 결과 선확정, reveal ID, 늦은 콜백 차단 불변식 유지.
- 같은 자동 룰렛 회차에서 두 명을 연속 공개해도 두 번째 원판이 반드시 정지하고 한 번만 완료됨.

## 12. 하위 호환성

- 기존 당첨 기록 localStorage 키와 CSV 형식을 유지한다.
- `sessionId`는 선택 필드로 추가해 과거 레코드도 계속 읽는다.
- `retto-roulette-pending-result` 버전 1은 공개 전 확정 결과 복구에만 사용하며 라이브 애니메이션 재개 상태로 해석하지 않는다.
- 과거 `mode: marble` 기록과 엔진 호환은 유지하되 현재 공개 선택지는 회전 룰렛·다트 복권으로 제한한다.
- 기존 카페 복사 파서와 명단 ID/가중치를 재사용한다.
- GitHub Pages 정적 배포 구조와 `/rettoroulette/` base 경로를 유지한다.
- 공개 연출 선택과 결과·복구 상태 구조가 바뀌는 호환 가능한 기능 변경이므로 minor 버전을 `0.10.0`으로 올린다.
